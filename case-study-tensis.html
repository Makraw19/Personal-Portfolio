01 — Context

Quality for AI is different.

Tensis ships AI-powered automation products used inside regulated workflows. That means the QA bar isn't just "does the feature work" — it's data integrity, security posture, performance at scale, and audit-ready traceability. All at once.

When I joined, the team had strong engineering but no formal QA strategy. My job: design it, own it, and scale it across a rapidly-shipping product roadmap.

02 — Strategy

Four pillars.

01 · Standards
Metrics & KPIs
Rigorous quality standards grounded in data integrity, security, accuracy, and regulatory compliance — measurable and visible.
02 · Risk
Proactive assessment
OWASP-driven security testing plus continuous risk identification and mitigation across every release path.
03 · Automation
Coverage at scale
Test automation framework expansion for data-driven scenarios — maximising reliable coverage without slowing releases.
04 · Resilience
Performance & load
Performance, load, and security testing for scalability and responsiveness, especially on critical data paths.
03 — Execution

From strategy to shipping.

  • Defect tracking overhauled. Clear triage, SLAs, owner accountability, and transparent resolution reporting.
  • 400+ API tests in Postman. Full multi-environment coverage (dev, staging, prod) for public APIs — reliable, scalable, enterprise-ready.
  • Playwright + MCP + Claude Code Pro. Currently building the next-gen web automation stack, pairing Playwright with the Model Context Protocol and Claude Code Pro for AI-assisted test authoring.
  • AI test suites. Navigated the ambiguity of testing AI products, designing large test suites that cover the UX surface around non-deterministic outputs.
  • Third-party integration testing. Thorough coverage for every downstream integration — the blast radius of a bad integration is the whole product.

04 — Takeaways

What AI QA demands.

Ambiguity is the work. AI outputs aren't deterministic, so the tests can't be either. You test the envelope — guardrails, fallbacks, the UX around the answer — not the answer itself.

Security is table stakes. OWASP assessments aren't a phase — they're a continuous discipline baked into every release.

Data integrity is the product. When your customers run their business on your output, every row has to be right, every time, and you have to prove it.

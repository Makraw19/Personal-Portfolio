01 — Context

A product every Australian clinic relies on.

Practice Cloud is PenCS's web platform for clinical analytics — used across Australia's Private Health Network to run reports on patient cohorts, MBS items, chronic disease management, and clinical audit. Hundreds of filters, thousands of data combinations, one answer that has to be right every time.

QA coverage was almost entirely manual. Every release meant a week of regression: a tester clicking through filter combinations, eyeballing totals, and flagging discrepancies. It worked, but it didn't scale — and it blocked every release.

02 — Diagnosis

Three bottlenecks.

  • Filter explosion. 400+ filters × multiple data shapes = combinations no human can test exhaustively.
  • Data-heavy validation. Each report compares aggregates across thousands of records — easy to get wrong by one row and never notice.
  • No reusable harness. Every test pass started from scratch because there was no framework to build on.
03 — Approach

A TestComplete framework built for data.

I designed a data-driven TestComplete framework that separated test intent from test data. Filters and expected results live in spreadsheets and CSVs; the framework runs every combination, validates the rendered output against the expected row, and reports a clean pass/fail matrix. 

// filter-suite.pjs  —  data-driven filter coverage
function testFilters() {
  var rows = DDT.ExcelDriver("data/filters.xlsx", "coverage");
  while (!rows.EOF()) {
    applyFilter(rows.Value("filter_name"), rows.Value("params"));
    var actual   = readTotal();
    var expected = rows.Value("expected_total");
    Log.Checkpoint("filter: " + rows.Value("filter_name"),
                   actual === expected);
    rows.Next();
  }
}

Critically, the framework was built to be extended: adding a new filter meant adding a row to a spreadsheet, not writing new code. The QA team could grow coverage without waiting on automation engineering.

05 — Takeaways

What I'd do again.

Data-driven from day one. Once data and logic are separated, coverage becomes a spreadsheet problem — and everyone on the team can contribute.

Build the framework, not just the tests. The tests that ship are less valuable than the harness that lets the team ship more of them after you leave.

Health data deserves extra care. In clinical analytics a wrong number isn't a bug — it's a trust problem. Validation has to go down to the row.
